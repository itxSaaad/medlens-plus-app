# Marketing copy voice

Guidelines for all MedLens+ marketing content (JSON, UI strings, meta descriptions).

## Tone

- Write like a helpful product person, not a compliance document.
- Use contractions naturally: you'll, don't, it's, we're.
- Lead with what the user gets, not what we avoid.
- Be specific: file names, visit prep, "LDL went from 118 to 142".

## Structure

- One safety disclaimer per page max (trust block or footer), not in hero or meta.
- Meta descriptions: unique per URL, under 155 characters, no "without diagnostic claims" boilerplate.
- Home FAQ: 4 questions only. Full FAQ lives on `/faq`.
- Max one em dash per section; prefer short sentences.

## SEO & links

- Hub-and-spoke internal links: blog → glossary → use case → how-it-works.
- Glossary terms: educational first; one soft CTA line at bottom.
- Use cases: scenario-led openings ("Your endocrinologist asks about A1c trend…").

## Words to vary

Avoid repeating on the same page:

- source-grounded
- doctor-ready
- without diagnosing
- safety-first (once per page is enough)

## Medical boundaries (still required)

- Never diagnose, prescribe, or suggest dosage changes.
- Use reference ranges from uploaded reports, not generic internet values.
- Encourage discussion with a qualified clinician when results matter.
