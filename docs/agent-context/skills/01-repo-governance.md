# Skill 01: Repo Governance

## Goal
Ensure every change aligns with branch strategy, PR policy, and quality gates.

## Steps
1. Confirm ticket scope from `planning/tickets/`
2. Keep change minimal and logically scoped
3. Use Conventional Commits
4. Verify branch naming policy is satisfied
5. Ensure PR checklist sections are complete

## Done When
- CI passes
- Review feedback addressed
- Docs updated for policy/process changes
