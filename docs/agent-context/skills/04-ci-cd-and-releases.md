# Skill 04: CI/CD and Releases

## Goal
Keep CI, release automation, and deployment gates stable and auditable.

## Rules
- CI is the quality gate for JS + Python + governance checks
- Releases are generated by `semantic-release` on `main`
- Tags follow `vX.Y.Z`
- Deploy workflow remains a controlled gate until enabled

## Validation
Run:
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`

Review:
- `.github/workflows/ci.yml`
- `.github/workflows/release.yml`
- `.github/workflows/deploy.yml`
- `.releaserc.json`
