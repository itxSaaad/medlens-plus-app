# Release Process

## Workflow Sequence
1. CI workflow validates quality and governance checks.
2. Release workflow runs only after successful CI for `main`.
3. Deploy workflow is currently a reserved skip-only gate.

## Release Automation Behavior
- Releases are generated by `semantic-release`.
- Version bumps follow Conventional Commits:
  - `feat:` => minor release
  - `fix:` => patch release
  - `feat!:` or `BREAKING CHANGE:` => major release
- Release tags are created as `vX.Y.Z`.
- GitHub Release notes are generated automatically.

## Requirements
- Conventional commits
- Semantic PR titles
- Green CI checks before merge to `main`
