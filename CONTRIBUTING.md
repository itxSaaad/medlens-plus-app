# Contributing to MedLens+

Thanks for contributing.

## Setup Expectations
- Read `README.md`
- Read `AGENTS.md`
- Pick one ticket from `planning/tickets/`
- Confirm scope before coding

## Workflow
1. Fork repository
2. Create branch: `feat/<ticket-id>-<slug>`
3. Implement small, reviewable commits
4. Add/adjust tests
5. Open PR using PR template

## Commit Convention
Use Conventional Commits:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` non-breaking restructure
- `test:` test changes
- `chore:` tooling/maintenance

## Code Review Rules
- No silent architecture deviations
- No medical-risky copy changes without safety review
- No schema changes without migration notes

## What To Work On
Start from:
- `planning/sprints/`
- `planning/tickets/`

## Community Conduct
By participating, you agree to `CODE_OF_CONDUCT.md`.
